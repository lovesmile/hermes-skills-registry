# Hermes Agent 四层存储体系

> 适用于：任何使用 Hermes Agent 的用户
> 目标：知道每个东西该放哪、不该放哪，不浪费 token，不污染上下文

---

## 一、四层存储总览

| 层级 | 容量 | 注入方式 | 存什么 | 典型寿命 |
|------|------|---------|--------|---------|
| **memory** | ~2,200字 | 每次对话自动注入 | 环境事实、工具特性、项目约定 | 跨会话（永久） |
| **user profile** | ~1,375字 | 每次对话自动注入 | 用户身份、偏好、沟通风格 | 跨会话（永久） |
| **skills** | 无上限（按需加载） | `skill_view(name)` 手动加载 | 可复用的流程、工作流、踩坑记录 | 跨会话（永久） |
| **session_search** | 无上限（FTS5检索） | 主动搜索 | 过去会话的完整对话记录 | 跨会话（自动保存） |

**内存（memory + user profile）是每次对话都注入的，所以容量有限，必须精打细算。**

---

## 二、各层存什么（正例）

### memory — 你的私人笔记

存：
- ✅ **环境事实**：操作系统、主机名、项目路径、已装工具
- ✅ **项目约定**：代码风格、命名规范、测试框架
- ✅ **工具特性**：某条命令的特殊参数、某工具的已知 bug
- ✅ **重复发生的问题**：每次部署都会遇到的坑

**格式**：声明性事实（`项目使用 pytest + xdist`），不是指令（`记得用 pytest -n 4`）

### user profile — 用户画像

存：
- ✅ **身份信息**：姓名、职业、角色
- ✅ **沟通偏好**：喜欢简洁还是详细、中文还是英文、专业术语还是口语
- ✅ **行为习惯**：做事风格（先问再干 / 直接干）、确认节奏
- ✅ **禁止项**：用户明确说过不想要的

### skills — 程序化记忆

存：
- ✅ **可复用的流程**：部署步骤、代码审查流程、测试策略
- ✅ **踩坑记录**：遇到过什么坑、怎么解决的、下次怎么避免
- ✅ **复杂任务的完整方案**：5次以上工具调用才完成的任务
- ✅ **用户的定制工作流**：用户偏好的一套做法

### session_search — 会话记录（自动维护）

存：
- ✅ **过去的全部对话**：自动保存，无需手动操作
- ✅ **已关闭的任务状态**：上次做到哪了、出了什么错

## 三、各层不存什么（反例）

### memory / user profile 不存

- ❌ **任务进度**：「PR #42 已提交」→ 7天后就过时了
- ❌ **完成日志**：「修复了 bug X」「Phase N done」→ 用 session_search
- ❌ **临时 TODO**：当前会话的任务列表 → 用 todo 工具
- ❌ **数字型状态**：bug编号、commit SHA、PR编号、文件数
- ❌ **7天内会过时的信息**：任何有保质期的事实

### skills 不存

- ❌ **用户个人偏好** → 存 memory / user profile
- ❌ **一次性任务记录** → 不用存，session_search 里有
- ❌ **当前会话的临时状态** → 会话上下文本身够用
- ❌ **其他技能已经覆盖的内容** → 先查重，别重复造轮

### session_search 不存

- ❌ 没有「不存」——它是自动的。但不要用它替代 memory 来存永久事实

## 四、空间节省技巧

### 4.1 memory 优化

```markdown
# ❌ 浪费（指令式、重复、啰嗦）
用户偏好：
- 希望输出直白，不用专业术语
- 希望数值要带文字说明
- 需要可执行指令格式
- 不喜欢分析建议

# ✅ 紧凑（声明式、合并同类项）
用户偏好输出：直白/无术语/可执行指令，不分析建议数值带说明
```

**核心口诀**：同类合并、去掉主语、去掉「希望/需要/要」、短横线改斜线

### 4.2 user profile 优化

```markdown
# ❌ 浪费
用户是软件开发工程师，主要使用 Python 和 TypeScript，住在上海，在上海工作
# ✅ 紧凑
Dev: Python/TS, 上海
```

**技巧**：
- 能用标签/分类就别写句子
- 同类事实放一行，用逗号/斜线分隔
- 上下文能推断的省略（如性别、年龄、城市）

### 4.3 分层策略：memory 存摘要，skills 存全文

```
memory（300字摘要）:
  项目A用pytest+ci流程。完整部署步骤→技能 deploy-A

skills（3000字全文）:
  部署项目A的完整7步流程，含踩坑记录和回滚方案
```

关键事实在 memory 中保留摘要和技能名，完整流程在 skill 里。

### 4.4 定期修剪

- 发现 MEMORY 中有某段超过2周没触发过 → 考虑删除
- 发现 user profile 中有重复条目 → 合并
- 发现 skills 中有过时的命令/路径 → 更新
- session_search 不需要手动维护

## 五、三问决策法

不知道该放哪的时候，问三个问题：

1. **7天后还有用吗？** → 没有 → 不用存（用 session_search 回溯）
2. **这是一个事实还是一个流程？** → 事实→ memory/user profile；流程→ skills
3. **有没有其他技能已经存了？** → 有 → 不重复存，更新原有技能

### 决策树

```
这个信息7天后还有用吗？
├── 没有 → 不用存（session_search 里有）
└── 有 →
    ├── 是用户个人偏好/环境事实 →
    │   ├── 是沟通风格/做事习惯 → user profile
    │   └── 是项目约定/工具特性 → memory
    └── 是可复用的流程/方法/踩坑记录 → skills
```

## 六、跨会话检索

**不要反复保存相同信息。** 过去会话的内容在 session_search 里：

```
session_search(query="部署项目A", limit=3)  # 搜到上次的部署日志
session_search(query="讨论过关于数据库的什么")  # 搜到相关的讨论记录
```

- **新事实** → 存 memory / user profile / skills
- **旧记录** → 搜 session_search

## 七、常见错误

| 错误 | 表现 | 纠正 |
|------|------|------|
| 把进度当事实存 | 「已提交 PR #42」「Phase 2 done」 | 用 session_search 查 |
| 指令式格式 | 「记得用 pytest -n 4」「每次先跑 lint」 | 改成声明式「项目用 pytest + xdist」 |
| 存冗余细节 | 完整复述一条命令的 5 个参数 | 存关键约束，完整步骤放 skills |
| 不清理过期 | 半年前的项目路径还在 memory 里 | 定期检查、删除过期条目 |
| 重复会话状态 | 当前会话已经说了 10 次的事又存进 memory | 先问自己：下次会话还需要吗？ |

## 八、审计脚本

技能附带 `scripts/audit.py`，一键扫描当前 memory/user profile 状态：

```bash
# 运行审计（默认 ~/.hermes/memories/）
python3 scripts/audit.py

# 指定路径
python3 scripts/audit.py /path/to/memories
```

输出内容：
- MEMORY.md / USER.md 各条目字数 vs 容量限制
- 问题检测：指令式表述、进度日志、陈旧条目、可迁移到 skills 的长内容
- 优化建议：合并方向、精简方案
- Skills 目录概览：各技能文件大小
